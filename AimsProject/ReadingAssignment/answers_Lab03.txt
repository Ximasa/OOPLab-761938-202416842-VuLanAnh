//Lab 03.14
Trong hệ thống AIMS, các lớp đóng vai trò là "Cái toàn thể" (Whole) và chứa các "Thành phần" (Part) thông qua mối quan hệ Tụ tập (Aggregation) bao gồm:

-Lớp Store và Cart: Cả hai lớp này đều là tập hợp của lớp Media. Một cửa hàng (Store) hay một giỏ hàng (Cart) sẽ quản lý một danh sách các đối tượng Media. Mối quan hệ này cho thấy các đối tượng Media có thể tồn tại độc lập bên ngoài vòng đời của Store hay Cart.
-Lớp CompactDisc (CD): Đây là lớp tụ tập của lớp Track. Một đĩa CD chứa một danh sách các bài hát/track.

Về việc khởi tạo: Tất cả các Constructor của các lớp toàn thể này (Store, Cart, CompactDisc) đều đã được kiểm tra để đảm bảo khởi tạo các danh sách tương ứng ngay khi đối tượng được tạo ra. Việc khởi tạo này giúp ngăn chặn lỗi NullPointerException khi thực hiện các thao tác thêm mới thành phần.

//Lab 3.15
If the passing object is not an instance of Media, what happens?
-> Trả lời:
Nếu đối tượng truyền vào không phải là một instance của Media, dòng code obj instanceof Media sẽ trả về false. Nhờ đó, hàm equals sẽ trả về false ngay lập tức thay vì cố gắng ép kiểu. Nếu chúng ta không kiểm tra instanceof mà trực tiếp ép kiểu (Media) obj, chương trình sẽ văng lỗi ClassCastException và bị crash.

1. What class should implement the Comparable interface? 
-> Trả lời:
Là lớp Media nên implement interface này để định nghĩa quy tắc sắp xếp mặc định cho tất cả sản phẩm.

2. In those classes, how should you implement the compareTo()method to reflect the ordering that we want?
-> Trả lời:
Trong Media, ta override hàm compareTo(Media other) và dùng logic so sánh title như đoạn code trong class MediaComparatorByTitleCost và MediaComparatorByCostTitle.

3. Can we have two ordering rules of the item (by title then cost and by cost then title) if we use this Comparable interface approach?  
-> Trả lời:
Không. Comparable chỉ cho phép định nghĩa một quy tắc sắp xếp duy nhất. Đây chính là lý do vì sao bài này yêu cầu dùng Comparator để có thể đổi qua lại giữa nhiều kiểu sắp xếp.

4. Suppose the DVDs have a different ordering rule from the other media types, that is by title, then decreasing length, then cost. How would you modify your code to allow this?
-> Trả lời:
Ta sẽ tạo một class DVDComparator riêng hoặc dùng tính năng Anonymous Inner Class khi gọi lệnh sort dành riêng cho danh sách DVD.
-> Đoạn code ví dụ:
Collections.sort(dvdList, (d1, d2) -> {
    int res = d1.getTitle().compareTo(d2.getTitle());
    if (res == 0) res = Integer.compare(d2.getLength(), d1.getLength()); 
    if (res == 0) res = Float.compare(d2.getCost(), d1.getCost()); 
    return res;
});
